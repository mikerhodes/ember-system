Migrate session store to Redis cluster

The current session store uses a single Postgres table which is becoming a bottleneck under load. Moving to a dedicated Redis cluster will reduce p99 latency on authenticated endpoints. See the architecture proposal and load test results for full context.

Work should follow the plan in INF-204. Once Redis is provisioned, the migration can proceed without downtime using the dual-write strategy.
